# RELIABILITY.md

## 목적

이 문서는 2026-06-11 1차 플레이어블 마스터를 안정적으로 만들기 위한 일정, 리소스, 엔진, 콘텐츠, 연출 제약 조건을 정리한다.

기획의 목표는 크지만, 한 달 안에 검증해야 하는 프로젝트이므로 모든 기능은 `구현 가능성`과 `마감 리스크`를 기준으로 관리한다.

마감일이 2026-06-11보다 앞당겨지면 아래 순서대로 범위를 줄인다.

| 컷 순서 | 항목 | 이유 |
| --- | --- | --- |
| 1 | Start Scene, End Scene | 핵심 전투 검증과 직접 관련이 낮음 |
| 2 | 고급 연출/카메라 컷 | 이펙터/디자이너 부재로 제작 비용이 큼 |
| 3 | 필살기 자동 승격 | 조건 판정, UI, 연출까지 함께 필요해 리스크가 큼 |

## 콘텐츠 수량 제한

### 직업

최초 마스터에서는 아래 3직업만 구현한다.

| 직업 | 역할 | 구현 우선 |
| --- | --- | --- |
| 검사 | 단일 공격/강화 | P0 |
| 마법사 | 광역 피해/필드 효과 | P0 |
| 힐러 | 회복/보호/버프 | P0 |

추가 직업은 마스터 이후 확장으로 분리한다.

### 카드

| 카드 분류 | 목표 수량 | 비고 |
| --- | --- | --- |
| 검사 전용 | 5장 | 공격, 강화, 단일 대상 중심 |
| 마법사 전용 | 5장 | 광역, 필드, 디버프 중심 |
| 힐러 전용 | 5장 | 회복, 보호, 버프 중심 |
| 공용 카드 | 5장 | 기본 공격/방어/유틸리티 |
| 특수 카드 | 약 5장 | 적 기믹 또는 필드 상호작용 |

저주 카드는 시스템 틀은 열어두되, 마감 상황에 따라 실제 카드 수를 줄인다.

카드 등급은 `Normal`, `Rare`, `Legendary`를 사용한다. 최초 마스터에서는 Legendary 카드를 많이 만들지 않는다. 전설 카드는 직업별 0~1장 수준으로 제한하고, 부족하면 Rare 카드로 대체한다.

### 몬스터

목표 수량은 5~7종이다.

권장 구성:

| 타입 | 수량 | 목적 |
| --- | --- | --- |
| 기본 공격형 | 1~2 | 전투 기본 학습 |
| 방어/버프형 | 1 | 우선 처치 판단 유도 |
| 디버프형 | 1 | 힐러/정화 가치 확인 |
| 광역 공격형 | 1 | 파티원별 방어도 압박 |
| 엘리트 후보 | 1 | 빌드 검증 |
| 보스 후보 | 1 | 스테이지 마무리 |

## 엔진 제약

### Unreal Engine 5.6.2

프로젝트는 UE 5.6.2 기준으로 구현한다.

기본 방향:

- C++ 또는 안정적인 코어 로직: 전투 상태, 행동 큐, 대상 판정, 효과 처리
- Blueprint: UI 연결, 빠른 프로토타입, 연출 트리거
- UMG: 카드 UI, 파티원 패널, 적 의도, 행동 큐 표시
- Data Asset/Data Table: 카드, 직업, 적, 상태이상, 보상 데이터
- Niagara: 가능한 경우 간단한 공격/필드/마법 효과
- Material: 단순 색상 변화, 강조, 피격, 버프 표현

보류:

- 복잡한 GAS 구조
- 고급 물리 연출
- 복잡한 시네마틱
- 온라인/멀티플레이
- 대규모 저장/로드 시스템

## 아트/연출 리소스 제약

현재 팀에는 전담 이펙터와 디자이너가 없다.

따라서 연출은 아래 기준으로 제한한다.

- 보유 에셋 우선 사용
- 무료/기존 에셋 재활용
- 간단한 Material 변경으로 상태 표현
- Niagara 튜토리얼 기반의 단순 이펙트만 사용
- 카드/필살기 연출은 카메라 컷보다 UI, 색상, 타격 정지, 화면 흔들림 위주로 처리
- 직업별 고유 연출은 최소 1개 핵심 이펙트만 목표

필살기 연출의 최소 기준:

- 일반 카드와 구분되는 화면 강조
- 전용 사운드 또는 임시 사운드
- 색상/이펙트 차이
- 짧은 타격 정지 또는 카메라 강조

전용 애니메이션 제작이 필요하면 마감 리스크로 분류한다.

## 시스템별 리스크

| 시스템 | 리스크 | 대응 |
| --- | --- | --- |
| 행동 큐 | 카드 순서, 대상, 상태가 꼬일 수 있음 | Battle Log와 큐 디버그 UI 구현 |
| 필살기 | 조건 판정, UI, 연출이 모두 필요함 | 마감 단축 시 추후 구현으로 이관 |
| 합동기 | 입력 패턴과 파티원 조건 조합이 복잡해질 수 있음 | MVP는 1~2개 규칙만 구현 |
| 스테이지 그리드 | 노드 생성과 보상 흐름이 늘어질 수 있음 | 고정형 그리드로 시작 |
| 카드 데이터 | 카드 수가 늘면 밸런싱 비용 증가 | 총 카드 수를 25장 내외로 제한 |
| UI | 파티원 3명 손패를 한 화면에 보여주기 어려움 | 선택 파티원 중심 UI로 제한 |
| 몬스터 | 7종 이상은 패턴 제작 부담 증가 | 5종 최소, 여유 있으면 7종 |

## 안정성 기준

마스터 빌드에서 반드시 안정적이어야 하는 항목:

- 전투 시작/종료가 정상 동작한다.
- 카드 사용 후 코스트가 정확히 감소한다.
- 행동 큐가 사용 순서대로 실행된다.
- 대상이 사망했을 때 전투가 멈추지 않는다.
- 적 턴이 정상적으로 넘어간다.
- 스테이지 클리어 시 스테이지 한정 상태가 초기화된다.
- 카드 보상이 덱에 정상 반영된다.

마스터 빌드에서 버그가 허용될 수 있는 항목:

- 임시 이펙트 품질
- 일부 카드 수치 밸런스
- 사운드 완성도
- UI 최종 폴리싱
- Start/End Scene 부재

## 의사결정 규칙

일정이 흔들릴 때는 아래 순서로 판단한다.

1. 전투 루프가 깨지는가?
2. 파티원별 개별 덱 구조가 유지되는가?
3. 공용 코스트와 행동 큐가 유지되는가?
4. 3직업이 모두 기본 역할을 수행하는가?
5. 스테이지 그리드가 최소 형태로 동작하는가?
6. 필살기/합동기는 마감 안에 감당 가능한가?
7. 연출은 보유 에셋과 간단한 Material/Niagara로 해결 가능한가?

위 질문에서 앞쪽 항목을 해치는 기능은 후순위로 뺀다.
