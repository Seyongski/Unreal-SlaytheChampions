# tech-debt-tracker.md

## 목적

이 문서는 기획과 구현 과정에서 의도적으로 남겨두는 기술적 부채, 임시 처리, 추후 개선 항목을 추적한다.

2026-06-11 1차 플레이어블 마스터까지는 완성도보다 핵심 전투 루프 검증을 우선한다. 따라서 일부 항목은 임시 구현을 허용하되, 반드시 이 문서에 기록한다.

## 임시 구현 허용 항목

마스터 전까지 아래 항목은 임시 구현을 허용한다.

- 카드/직업/몬스터 데이터 일부 하드코딩
- 임시 UI 레이아웃
- 임시 사운드
- 단순 Niagara/Material 이펙트
- 고정형 스테이지 그리드
- 몬스터 AI 고정 패턴
- 포션 효과 단순화

단, 아래 항목은 임시 구현이라도 전투를 깨면 안 된다.

- 카드 사용 후 코스트 처리
- 행동 큐 실행 순서
- 대상 사망 처리
- 전투 종료 판정
- 스테이지 클리어 시 상태 초기화
