# damage-logic.md

## 목적

이 문서는 카드 이벤트 큐 기반 전투에서 피해, 방어, 대상, 행동 순서를 계산하는 기본 규칙을 정의한다.

## 핵심 구조

각 파티원은 개별 덱과 손패를 가진다. 플레이어는 파티원을 선택해 해당 파티원의 손패를 확인하고, 파티 전체가 공유하는 코스트를 소비해 카드를 사용한다.

카드는 전투 중 즉시 결과를 만드는 대신, 기본적으로 `Action Event`를 생성한다.

```text
카드 사용
-> Action Event 생성
-> 플레이어 행동 큐에 등록
-> 턴 종료
-> 큐 순서대로 행동 실행
-> 피해/방어/상태이상 처리
```

단, 카드 설계상 즉시 처리되어야 하는 드로우, 에너지 회복, 카드 생성 등은 `Instant Effect`로 분류할 수 있다.

## 덱과 코스트 규칙

| 항목 | 규칙 |
| --- | --- |
| 덱 | 파티원별 개별 덱 |
| 손패 | 파티원별 개별 손패 |
| 코스트 | 파티 전체 공용 코스트 |
| 카드 사용 | 선택한 파티원의 손패에서 사용 |
| 행동 주체 | 기본적으로 해당 카드를 보유한 파티원 |
| 행동 큐 | 모든 파티원의 사용 카드가 하나의 플레이어 행동 큐에 합쳐짐 |

예시:

```text
공용 코스트 3
수호자 카드 1장 사용: 코스트 1
척후병 카드 1장 사용: 코스트 2
남은 코스트 0
턴 종료 후 수호자 이벤트 -> 척후병 이벤트 순서로 실행
```

## 카드 분류

카드는 `분류`와 `등급`을 별도 축으로 가진다.

- 분류: 카드가 어떤 사용 범주에 속하는가
- 등급: 카드가 얼마나 강하고 희소한가

### 카드 분류

| 분류 | 설명 | 사용 기준 |
| --- | --- | --- |
| Common Card | 모든 직업이 사용할 수 있는 공용 카드 | 기본 공격, 기본 방어, 단순 유틸리티 |
| Class Card | 특정 직업 또는 파티원 역할 전용 카드 | 수호자 방어, 암살자 연타, 사제 회복 |
| Curse Card | 덱과 전투 선택지를 방해하는 카드 | 사용 불가, 코스트 낭비, 피해 유발 |
| Special Card | 특정 적 기믹, 이벤트, 보스전에서 사용 가능한 카드 | 보스 장치 해제, 필드 상호작용 |

추후 카드 분류는 추가될 수 있다. 단, 새 분류는 기존 분류로 표현할 수 없는 플레이 목적이 있을 때만 추가한다.

### 카드 등급

| 등급 | 설명 | 설계 기준 |
| --- | --- | --- |
| Normal | 기본 등급 카드 | 단순하고 안정적인 효과, 덱의 뼈대 |
| Rare | 희귀 카드 | 조건부 시너지, 강한 효율, 빌드 방향 제시 |
| Legendary | 전설 카드 | 런의 전략을 바꿀 수 있는 강한 효과 또는 고유 기믹 |

등급은 카드 보상 확률, 상점 가격, 강화 가치, 덱 빌드 방향에 영향을 준다.

MVP에서는 전설 카드를 많이 만들지 않는다. 각 직업별 0~1장 수준으로 제한하고, 필요하면 Phase 5 이후 확장한다.

카드 수치 밸류는 `QUALITY_SCORE.md`의 카드 밸류 기준을 따른다.

## 이벤트 타입

| 타입 | 설명 | 예시 |
| --- | --- | --- |
| Attack | 피해를 준다 | 단일 공격, 광역 공격, 연타 |
| Guard | 방어도를 얻는다 | 자신 방어, 아군 보호, 파티 방어 |
| Skill | 전투 보조 효과 | 카드 드로우, 에너지 회복, 카드 강화 |
| Status | 상태이상을 부여한다 | 취약, 약화, 표식, 기절 |
| Support | 아군을 강화한다 | 힘 증가, 보호막, 다음 공격 강화 |
| Special | 캐릭터 고유 규칙 | 반격 태세, 소환, 위치 변경 |
| Co-op | 여러 파티원의 조건으로 발동한다 | 합동 공격, 연계 방어 |
| Ultimate | 조건 충족 시 강화 연출과 큰 효과를 낸다 | 필살 공격, 대형 버프, 대형 디버프 |

## 기본 피해 공식

```text
최종 피해 = max(0, 기본 피해 + 공격자 공격 보정 + 카드 보정 - 대상 방어 보정)
```

초기 프로토타입에서는 계산식을 단순하게 유지한다.

```text
카드 피해 = 카드 기본값 + 파티원 Power
실제 체력 피해 = max(0, 카드 피해 - 대상 Block)
```

## 방어 규칙

```text
획득 방어도 = 카드 기본 방어 + 파티원 Guard Power
```

기본값:

- 방어도는 파티원별로 보유한다.
- 턴 종료 정리 단계에서 일반 방어도는 0으로 초기화된다.
- 특정 카드나 상태는 방어도 유지 규칙을 예외로 만들 수 있다.
- 카드 효과로 모든 파티원의 방어도를 올릴 수 있다.
- 카드 효과로 특정 파티원이 아군이 받을 피해를 대신 받을 수 있다.

## 피해 대신 받기 규칙

피해 대신 받기는 파티원별 방어도를 유지하면서도 탱커 역할을 만들기 위한 예외 규칙이다.

```text
피해 대상 확인
-> 피해 대신 받기 효과 확인
-> 대신 받을 파티원으로 피해 대상 변경
-> 대신 받는 파티원의 Block과 상태를 기준으로 피해 계산
```

동시에 여러 대신 받기 효과가 있을 경우 우선순위는 아래 순서를 따른다.

1. 카드에 명시된 전용 보호 효과
2. 가장 최근에 적용된 보호 효과
3. 체력이 가장 높은 보호자

## 대상 규칙

카드는 아래 대상 타입 중 하나를 가진다.

| 대상 타입 | 설명 |
| --- | --- |
| Self | 행동하는 파티원 자신 |
| All Allies | 모든 아군 |
| Target Ally | 선택한 아군 1명 |
| Target Enemy | 선택한 적 1마리 |
| All Enemies | 모든 적 |
| All Units | 모든 아군과 모든 적 |
| Field | 필드 또는 전장 상태 |
| Random Enemy | 무작위 적 1마리 |
| Lowest HP Enemy | 체력이 가장 낮은 적 |
| Marked Target | 표식이 있는 대상 우선 |

## 행동 주체 규칙

카드 이벤트는 반드시 `행동 주체`를 가진다.

기본 규칙:

- 카드는 파티원 개별 덱에 들어간다.
- 카드를 사용한 파티원이 행동 주체가 된다.
- 공용 카드는 여러 직업 덱에 들어갈 수 있지만, 실제 행동 주체는 그 카드를 사용한 파티원이다.
- 직업 전용 카드는 해당 직업 또는 파티원만 덱에 넣을 수 있다.
- 저주 카드는 행동 주체가 없거나, 보유 파티원에게 부정적 효과를 준다.
- 특수 카드는 적 기믹 또는 필드 조건이 맞을 때만 사용 가능할 수 있다.

## 실행 순서

플레이어 행동 큐는 카드 사용 순서대로 실행된다.

```text
1번 카드 이벤트
-> 2번 카드 이벤트
-> 3번 카드 이벤트
-> 플레이어 행동 종료
-> 적 AI 행동 시작
```

예외 규칙:

- `Fast`: 큐 앞쪽으로 이동한다.
- `Delayed`: 다음 턴 시작 시 실행된다.
- `Reaction`: 조건 충족 시 적 턴 중 끼어든다.
- `Combo`: 직전 이벤트 타입을 참조한다.

예외 규칙은 재미를 만들 수 있지만 복잡도를 크게 올리므로, MVP에서는 `Combo` 1종만 제한적으로 사용한다.

## 협동 기믹 규칙

협동 기믹은 여러 파티원의 카드 사용, 대상 공유, 상태 조건이 결합될 때 발동하는 특수 이벤트다.

협동 기믹은 아래 조건 중 하나 이상을 요구한다.

| 조건 타입 | 설명 | 예시 |
| --- | --- | --- |
| Multi Actor | 서로 다른 파티원이 같은 턴에 조건 카드를 사용 | 수호자가 보호 후 검사가 공격 |
| Shared Target | 여러 파티원이 같은 대상을 지정 | 표식 대상에게 2명 이상 공격 |
| Sequence | 정해진 순서로 카드 사용 | 방어 -> 강화 -> 공격 |
| State Match | 특정 상태나 버프가 있는 상태에서 카드 사용 | 강화 상태의 검사 공격 |
| Field Match | 특정 필드 효과가 있을 때 카드 사용 | 화염 지대에서 폭발 카드 사용 |

협동 기믹은 기본 카드 이벤트를 대체하거나, 추가 이벤트를 큐에 삽입할 수 있다.

```text
카드 A 이벤트
-> 카드 B 이벤트
-> 협동 조건 확인
-> Co-op Event 추가 또는 기존 이벤트 강화
```

## 필살기 규칙

필살기는 기존 카드 효과가 크게 강화된 상위 이벤트다. 효과 방향은 큰 데미지, 큰 버프, 큰 디버프, 광역 제어 등으로 설계할 수 있다.

필살기는 반드시 별도 사용조건과 별도 연출 조건을 가진다.

| 항목 | 규칙 |
| --- | --- |
| 효과 | 기존 카드 효과의 강화판 |
| 조건 | 특정 카드 조합, 상태 누적, 행동 순서, 대상 조건 |
| 연출 | 일반 카드와 구분되는 카메라, 이펙트, 애니메이션, 사운드 |
| 비용 | 공용 코스트를 쓰거나, 조건 충족 자체를 비용으로 사용 |
| 판정 | 이벤트 실행 시점에 조건을 확인 |
| 발동 | 조건을 만족한 카드 이벤트가 자동으로 Ultimate Event로 승격 |

기본 발동 방식은 `조건 충족 후 해당 이벤트가 실행될 때 자동으로 필살기로 승격`되는 방식이다. 플레이어가 별도 필살기 버튼을 누르지 않는다.

```text
1. 검사 카드: 강화 사용
2. 검사 카드: 강화 사용
3. 검사 카드: 강화 사용
4. 검사 카드: 공격 사용
5. 턴 종료
6. 공격 이벤트 실행 시 강화 3스택 확인
7. 일반 공격이 검사 필살기 이벤트로 승격
```

## 필살기 조건 데이터

필살기 조건은 전투 로그와 상태 스택을 참조한다.

| 데이터 | 설명 |
| --- | --- |
| Actor | 어떤 파티원이 조건을 만족해야 하는가 |
| Required Card | 어떤 카드 또는 카드 태그가 필요한가 |
| Required Count | 몇 번 사용해야 하는가 |
| Required State | 어떤 버프/디버프/상태가 필요한가 |
| Required Target | 특정 대상 조건이 필요한가 |
| Timing Window | 기본값은 같은 스테이지 내부 유지 |
| Consume Rule | 발동 후 스택이나 상태를 소비하는가 |

## 필살기 예시: 검사 강화 참격

```text
캐릭터: 검사
조건 카드: 강화
조건: 같은 스테이지 안에서 강화 3스택 보유
발동 카드: 공격 타입 카드
대상: Target Enemy
효과: 기존 공격 피해를 크게 강화
추가 효과 후보: 방어도 일부 무시, 취약 부여, 광역 충격파
소비 규칙: 발동 후 강화 스택 3개 소비
유지 규칙: 필살기 조건 스택은 스테이지 내부 유지, 스테이지 클리어 시 초기화
연출: 전용 카메라 컷, 검기 이펙트, 타격 정지
```

## 필살기 밸런스 원칙

- 필살기는 일반 카드보다 강해야 하지만, 매 턴 반복 발동되면 안 된다.
- 조건은 플레이어가 전투 중 의식하고 준비할 수 있어야 한다.
- 조건 UI는 현재 진행도를 보여줘야 한다.
- 필살기 발동이 항상 정답이 되지 않도록 코스트, 타이밍, 대상 선택의 고민을 남긴다.
- 필살기는 캐릭터 정체성과 직업 전용 카드의 가치를 강화해야 한다.

## 예시 카드 처리

```text
카드: 방패 올리기
주체: 수호자
타입: Guard
대상: Self
효과: Block 8 획득
```

```text
카드: 찌르기
주체: 척후병
타입: Attack
대상: Enemy
효과: 피해 6
```

사용 순서:

```text
방패 올리기 -> 찌르기 -> 턴 종료
```

실행 결과:

```text
수호자 Block 8 획득
척후병이 선택한 적에게 피해 6
적 AI 행동
```

## 미정 항목

- Power, Guard Power 같은 스탯을 파티원별로 둘지 카드 태그별로 둘지
- 공격 이벤트가 대상 사망 시 자동으로 다른 적을 재지정하는지
- 카드 이벤트 큐를 UI에서 얼마나 상세히 보여줄지
- 공용 카드를 모든 직업 덱에 자유롭게 넣을 수 있는지, 직업별 허용 풀이 있는지
- 특수 카드가 보상으로 획득되는지, 전투 중 임시 생성되는지
- 필살기 조건 UI를 파티원 초상화, 카드, 행동 큐 중 어디에 표시할지
