# Interface & Camera

## 역할

플레이어에게 전투 정보를 보여주고, 조작 입력을 각 시스템에 전달하며, 피드백과 카메라 연출을 담당한다.

카드 조작 규칙은 `Card & Deck Engine`이 소유한다. 이 모듈은 카드 데이터나 사용 가능 여부를 직접 판단하지 않고, 손패/코스트/상태/대상 정보를 표시하고 입력 이벤트를 전달한다.

## 담당 범위

- Battle HUD
- Visual Feedback
- Meta UI
- 파티원 선택 표시
- 선택 파티원의 손패 표시
- 공용 코스트, 상태이상, 적 의도 표시
- 카드 선택/대상 선택 입력 전달
- 합동기/필살기 카메라 워킹

## 직접 처리하지 않는 것

- 카드 사용 가능 여부 계산
- 카드 코스트 차감
- 카드 사용 후 discard pile 이동
- 드로우/버림/셔플 규칙
- 카드 Action Event 생성
- 보상 카드 선택 결과 적용

위 책임은 `Card & Deck Engine`과 `Combat Kernel`에서 처리한다.

## 하위 시스템

| 하위 시스템 | 내용 | 완료 기준 |
| --- | --- | --- |
| Battle HUD | 손패, 공용 코스트, 파티원 상태, 적 의도 표시 | 플레이어가 현재 선택 가능한 정보를 읽을 수 있다 |
| Input Relay | 파티원 선택, 카드 클릭, 대상 선택 입력을 담당 시스템에 전달 | UI가 규칙을 직접 계산하지 않고 요청만 보낸다 |
| Visual Feedback | 타겟팅 연출, 데미지 로그, 합동기/필살기 카메라 | 카드 사용 결과가 시각적으로 구분된다 |
| Meta UI | 스테이지 맵, 덱 요약, 보상 선택 화면 | 전투 밖 진행을 UI로 조작할 수 있다 |

## Phase별 목표

| Phase | 목표 |
| --- | --- |
| Phase 1 | 전투 HUD, 파티원 선택 표시, 손패 표시, 카드/대상 입력 전달, 턴 종료 버튼 |
| Phase 2 | 3인 파티 UI, 상태이상/버프 표시, 합동기/필살기 피드백 |
| Phase 3 | 스테이지 맵, 보상 선택, 덱 제거 UI |
| Phase 4 | 런 상태, 유물/덱 요약, 보스 경고 표시 |
| Phase 5 | 최소 연출, 카메라, 사운드/이펙트 마감 |
