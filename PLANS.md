# PLANS.md

## Module Work Breakdown

팀 작업 분배와 담당 파트 기준은 `docs/module-work-breakdown.md`를 기준으로 관리한다.

Phase는 일정과 마감 단위로 유지하되, 실제 작업은 아래 6개 모듈 오너십으로 나누어 진행한다.

- Combat Kernel
- Card & Deck Engine
- Entity & Stats
- NPC Brain
- World Progress
- Interface & Camera

GitHub Issue와 Notion 작업 보드의 작업 제목은 `[Module][Phase] 작업명` 형식을 권장한다.

## 목적

이 문서는 파티 기반 로그라이크 덱빌딩 게임의 개발 마일스톤, MVP 범위, 우선순위, 보류 항목을 정리한다.

현재 프로젝트의 핵심 방향은 다음과 같다.

```text
파티원별 개별 덱
-> 파티 공용 코스트 사용
-> 카드 사용 순서대로 행동 큐 생성
-> 조건 충족 시 필살기 자동 승격
-> 플레이어 행동 후 적 AI 행동
-> 그리드 스테이지 진행
```

## 개발 기준

우선순위는 아래 순서로 판단한다.

1. 핵심 전투 루프를 검증하는가?
2. 파티원별 개별 덱과 공용 코스트 구조를 보여주는가?
3. 행동 큐와 자동 승격 필살기가 실제 재미로 이어지는가?
4. UE5에서 빠르게 프로토타입 가능한가?
5. 이후 콘텐츠 확장에 재사용 가능한가?

### 완료 기준

- `core-loop.md`에 전투 루프가 정리되어 있다.
- `pillars.md`에 3대 핵심 재미가 정리되어 있다.
- `damage-logic.md`에 카드, 대상, 방어, 필살기, 합동기 규칙이 정리되어 있다.
- `level-rules.md`에 스테이지 그리드와 유지/초기화 범위가 정리되어 있다.
- `gdd-template.md`에 신규 시스템 작성 템플릿이 있다.

### 현재 상태

완료.

기본 문서 레이아웃, 핵심 전투 구조, MVP 범위, UE5 아키텍처, 리소스 제약, 품질 기준, 제품 방향, 세계관 톤, 장르 문법, 기술 부채 추적 문서까지 1차 작성이 끝났다.

다음 단계는 Phase 1 전투 프로토타입이다.

## Phase 1: 전투 프로토타입

### 목표

가장 작은 전투 화면에서 “이 게임이 재미있을 수 있는지”를 확인한다.

### 포함 범위

- 파티원 2명
- 적 1~2마리
- 파티원별 개별 손패
- 캐릭터별 덱 1~10장
- 캐릭터별 기본 드로우 5장
- 드로우 pile 부족 시 discard pile 재순환
- 파티 공용 코스트
- 파티원 선택 UI
- 카드 사용 후 행동 큐 등록
- 턴 종료 후 행동 큐 순차 실행
- 적 AI 기본 행동
- 파티원별 방어도
- 단일 대상 공격, 자기 방어, 아군 전체 방어

### 제외 범위

- 전체 스테이지 그리드
- 상점, 이벤트, 휴식
- 복잡한 보스 패턴
- 다수의 유물
- 완성형 연출
- 저장/로드

### 완료 기준

- 플레이어가 파티원을 선택해 해당 파티원의 카드를 사용할 수 있다.
- 공용 코스트가 모든 파티원 카드 사용에 공유된다.
- 카드 사용 순서가 행동 큐에 반영된다.
- 턴 종료 후 행동 큐가 순서대로 실행된다.
- 적이 플레이어 행동 후 기본 AI에 따라 행동한다.
- 방어도가 파티원별로 적용된다.

## Phase 2: MVP 전투 빌드

### 목표

프로토타입 전투를 MVP 수준의 반복 가능한 전투 경험으로 확장한다.

### 포함 범위

- 파티원 3명 기준 전투
- 적 최대 3마리
- 공용 카드, 직업 전용 카드, 저주 카드, 특수 카드의 기본 구조
- 카드 대상 타입 전체 적용
  - Self
  - Target Ally
  - All Allies
  - Target Enemy
  - All Enemies
  - All Units
  - Field
- 상태이상 4~6종
- 합동기 1~2종
- 필살기 1인당 1종
- 필살기 자동 승격
- 필살기 조건 스택의 스테이지 내부 유지 규칙
- 전투 보상 카드 선택

### 최초 직업 3종

| 직업 | 역할 | 덱 방향 | 필살기 방향 |
| --- | --- | --- | --- |
| 검사 | 단일 공격/강화 | 강화 누적 후 강한 공격 | 강화 3스택 후 공격 자동 승격 |
| 마법사 | 광역 피해/필드 효과 | 원소반응, 필드, 광역 공격 | 원소/필드 조건 충족 후 광역 폭발 또는 대형 디버프 |
| 힐러 | 회복/보호/버프 | 회복, 정화, 아군 강화 | 누적 회복 또는 보호 조건 후 대형 회복/파티 버프 |

최초 마스터에서는 위 3직업만 구현 대상으로 삼는다. 이후 직업은 Phase 5 이후 콘텐츠 확장으로 추가한다.

### 완료 기준

- 3명의 파티원이 각자 다른 덱과 역할을 가진다.
- 공용 코스트 때문에 어느 파티원의 카드를 쓸지 선택 압박이 생긴다.
- 합동기가 적어도 1개 이상 실제 전투에서 발동된다.
- 필살기가 자동 승격으로 발동된다.
- 전투 3~5회를 반복해도 루프가 이해 가능하다.

## Phase 3: 스테이지 그리드

### 목표

전투 하나가 아니라, 한 스테이지 안에서 선택과 누적이 이어지는 구조를 만든다.

### 포함 범위

- 그리드 노드 선택
- 일반 전투 노드
- 이벤트 노드
- 휴식 노드
- 엘리트 노드
- 보스 노드
- 전투 보상
- 스테이지 내부 상태 유지
- 스테이지 클리어 시 필살기 조건 스택 초기화

### 완료 기준

- 플레이어가 그리드에서 다음 노드를 선택할 수 있다.
- 노드 결과가 전투/보상/회복/이벤트로 연결된다.
- 필살기 조건 스택이 같은 스테이지 안에서 유지된다.
- 스테이지 클리어 시 스테이지 한정 상태가 초기화된다.

## Phase 4: 런 구조

### 목표

여러 스테이지를 연결해 로그라이크 덱빌딩의 런 구조를 만든다.

### 포함 범위

- Act 1 구성
- 일반전 6종
- 엘리트 2종
- 보스 1종
- 카드 보상 풀
- 유물 10~15종
- 카드 제거/강화 수단
- 상점
- 휴식
- 이벤트 5종 이상

### 완료 기준

- Act 1을 처음부터 끝까지 플레이할 수 있다.
- 전투 보상이 다음 전투 선택에 영향을 준다.
- 덱이 런 중 실제로 변한다.
- 플레이어가 파티 조합과 카드 보상 사이에서 선택하게 된다.

## Phase 5: 콘텐츠 확장

### 목표

MVP 구조가 검증된 뒤 콘텐츠 양과 변주를 늘린다.

### 확장 후보

- 신규 직업
- 직업 전용 카드 추가
- 합동기 추가
- 필살기 조건 다양화
- 보스별 특수 카드
- 저주 카드 풀 확장
- 유물/장비 추가
- 이벤트 분기
- 난이도 단계
