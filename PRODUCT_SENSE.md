# PRODUCT_SENSE.md

## 목적

이 문서는 게임의 타겟 유저, 시장 매력, 차별점, 최초 마스터에서 보여줘야 할 세일즈 포인트를 정리한다.

## 한 문장 피치

짧게 즐길 수 있는 파티 기반 로그라이크 덱빌딩 게임으로, 각 파티원의 개별 덱을 조합해 전투를 설계하고 조건을 만족하면 필살기 연출로 전투 판도를 뒤집는다.

## 타겟 유저

### 핵심 타겟

- 덱빌딩 로그라이크를 좋아하는 유저
- 긴 플레이 시간을 내기 어렵지만 깊이 있는 선택을 원하는 유저
- 한 판을 가볍게 시작하고, 짧은 세션 안에서 빌드 조합의 재미를 느끼고 싶은 유저
- 단일 캐릭터 덱빌딩보다 파티 조합과 역할 분담을 좋아하는 유저
- 카드 시너지, 조건부 발동, 전투 중 판도 변화에 재미를 느끼는 유저

### 플레이 상황

- 10~30분 단위로 가볍게 플레이
- 매번 다른 카드/파티 조합을 시도
- 깊게 파고들면 빌드 최적화가 가능하지만, 처음에는 직관적으로 이해 가능해야 함

## 핵심 매력

### 1. 파티원별 개별 덱

기존 덱빌딩 게임은 보통 하나의 덱을 중심으로 플레이한다. 이 게임은 검사, 마법사, 힐러가 각자 개별 덱을 가지고 있고, 플레이어는 공용 코스트를 누구의 카드에 쓸지 결정한다.

이 구조는 매 턴 아래 질문을 만든다.

- 이번 턴은 검사의 공격에 투자할까?
- 마법사의 광역/필드 카드로 판을 깔까?
- 힐러의 회복/보호 카드로 다음 턴을 준비할까?

### 2. 파티 시너지로 생기는 조합 다양성

파티원 각자의 카드가 독립적으로 존재하지만, 실제 전투에서는 서로의 카드가 한 행동 큐 안에서 연결된다.

예시:

```text
힐러가 보호 버프 부여
-> 마법사가 필드 효과 생성
-> 검사가 강화 공격
-> 조건 충족 시 필살기 자동 승격
```

이 조합 구조는 같은 직업 3개로 시작해도 다양한 전투 플랜을 만들 수 있다.

### 3. 덱빌딩에서 보기 힘든 필살기 연출

필살기는 단순한 큰 데미지가 아니라, 플레이어가 조건을 쌓고 특정 타이밍에 터뜨리는 전투 고점이다.

필살기의 역할:

- 전투 판도 전환
- 캐릭터 정체성 강화
- 카드 조합의 보상
- 짧은 세션 안에서 강한 손맛 제공

필살기는 자동 승격 방식이므로, 플레이어가 별도 버튼을 누르는 것보다 “내가 만든 순서가 터졌다”는 감각을 준다.

## 차별화 포인트

| 요소 | 일반 덱빌딩 로그라이크 | 이 프로젝트 |
| --- | --- | --- |
| 덱 구조 | 보통 단일 덱 중심 | 파티원별 개별 덱 |
| 코스트 | 한 캐릭터의 자원 | 파티 전체 공용 코스트 |
| 전투 설계 | 카드 1장 단위 선택 | 파티원 선택 + 카드 순서 설계 |
| 시너지 | 카드 간 효과 조합 | 파티원 역할, 카드, 행동 큐 조합 |
| 고점 연출 | 강한 카드/유물 중심 | 조건 충족 필살기 자동 승격 |
| 세션 감각 | 긴 런 중심 | 짧은 세션에서도 판도 변화 제공 |

## 최초 마스터에서 반드시 보여줘야 할 재미

2026-06-11 마스터에서 아래 경험은 반드시 확인되어야 한다.

- 검사, 마법사, 힐러가 서로 다른 역할로 느껴진다.
- 플레이어가 공용 코스트를 누구에게 쓸지 고민한다.
- 카드 사용 순서가 행동 큐에 반영된다.
- 파티원 카드 조합으로 예상한 결과가 나온다.
- 필살기 또는 그 대체 구조가 전투의 고점으로 느껴진다.
- 한 스테이지 안에서 조건을 쌓고 중요한 전투에서 터뜨리는 재미가 있다.

마감이 앞당겨져 필살기를 추후 구현으로 빼는 경우에도, 행동 큐와 파티원별 덱 구조는 반드시 살아 있어야 한다.

## 마케팅 키워드

- 파티 기반 덱빌딩
- 짧게 즐기는 로그라이크
- 개별 덱 파티 전투
- 자동 승격 필살기
- 카드 순서가 만드는 전투 연출
- 검사/마법사/힐러 조합
- 공용 코스트 전략
- 스테이지 안에서 쌓는 필살기 조건

## 스팀/소개문 후보

### 짧은 소개

검사, 마법사, 힐러의 개별 덱을 조합해 전투를 설계하는 파티 기반 로그라이크 덱빌딩 게임.

### 조금 긴 소개

각 파티원은 자신만의 덱을 가지고, 파티는 하나의 공용 코스트를 공유한다. 누구의 카드를 어떤 순서로 사용할지 결정하고, 조건을 만족시키면 카드 이벤트가 필살기로 자동 승격되어 전투의 흐름을 뒤집는다.

## 시장 리스크

| 리스크 | 설명 | 대응 |
| --- | --- | --- |
| 복잡도 장벽 | 파티원별 덱이 초반에 어렵게 느껴질 수 있음 | 최초 직업 3개 역할을 명확히 분리 |
| 긴 플레이 부담 | 타겟 유저는 많은 시간을 쓰기 어려움 | 짧은 스테이지와 빠른 전투 템포 유지 |
| 필살기 과의존 | 필살기가 너무 강하면 일반 카드가 약해짐 | 조건 스택, 자동 승격, 소비 규칙으로 제한 |
| 연출 기대치 | 필살기라는 이름이 큰 연출 기대를 만들 수 있음 | 보유 에셋, Material, Niagara 기반의 명확한 최소 연출 확보 |
| 차별점 약화 | 단순 파티 덱빌딩만으로는 익숙하게 보일 수 있음 | 공용 코스트와 행동 큐, 자동 승격 필살기를 전면에 둠 |

## 제품 판단 기준

새 기능이나 콘텐츠를 추가할 때 아래 질문에 답해야 한다.

1. 짧은 세션 안에서도 재미가 드러나는가?
2. 검사/마법사/힐러 중 하나의 역할을 더 선명하게 만드는가?
3. 파티원별 개별 덱 구조를 활용하는가?
4. 공용 코스트 선택을 더 흥미롭게 만드는가?
5. 행동 큐와 카드 순서의 의미를 강화하는가?
6. 필살기 또는 합동기의 전투 설계감을 강화하는가?
7. UI와 설명이 복잡해지지 않는가?

## 피해야 할 방향

- 한 판이 너무 길어지는 구조
- 파티원별 덱이 사실상 하나의 덱처럼 느껴지는 구조
- 필살기 조건이 너무 어려워 거의 보이지 않는 구조
- 필살기만 기다리는 게임이 되는 구조
- 카드 텍스트가 길어져 가볍게 즐기기 어려워지는 구조
- 파티 조합보다 특정 카드 한 장의 성능만 중요한 구조
